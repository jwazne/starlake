## Create a new project

To create a new project, you need first to create an empty folder and run the
starlake bootstrap CLI command from there:

<Tabs groupId="platforms">
<TabItem value="linux_macos" label="Linux/MacOS">

```sh
$ mkdir $HOME/myproject
$ cd $HOME/myproject
$ $HOME/starlake/starlake.sh bootstrap
```

</TabItem>
<TabItem value="windows" label="Windows">

```powershell
c:\> mkdir %userprofile%\myproject
c:\users\me\myproject> cd %userprofile%\myproject
c:\users\me\myproject> %userprofile%\starlake\starlake.cmd bootstrap
```

</TabItem>
</Tabs>

:::note

By default, theproject will be created in the current working directory.
To bootstrap the project in a different folder set COMET_ROOT env variable to that folder:

<Tabs groupId="platforms">
<TabItem value="linux_macos" label="Linux/MacOS">

```
$ COMET_ROOT=/my/other/project $HOME/starlake/starlake.sh bootstrap
```

</TabItem>
<TabItem value="windows" label="Windows">

```
c:\> export COMET_ROOT=c:\my\other\project
c:\> %userprofile%\starlake\starlake.cmd bootstrap
```

</TabItem>
</Tabs>

:::

## Understanding the project hierarchy

- The `diagrams` folder holds all the entity/relations and access control views you will be able to interactively create using Starlake WYSIWYG tooling.
- The `incoming` folder host al the files you will want to load into your warehouse. This is explained in more detail in the [quickstart chapter](100.quickstart.mdx)
- The `metadata` folder contains the extract, load and transform jobs Starlake will apply to incoming files.
